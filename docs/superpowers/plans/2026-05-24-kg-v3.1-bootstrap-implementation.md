# kg v3.1 Implementation Plan — Bootstrap + GitHub Releases

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.3.1 of the kg plugin so end users can install without a Go toolchain or CGO build. Two parts shipped together: (1) a GoReleaser + GitHub Actions matrix that publishes prebuilt `.tar.gz` artifacts for darwin/arm64, darwin/amd64, linux/amd64, linux/arm64 on every `v*` tag; (2) a `bootstrap.sh` + smart SKILL pre-check that auto-locates `kg`, offers via `AskUserQuestion` to download the matching release, verifies SHA-256, and extracts into `${KG_HOME:-$HOME/.config/kg}/`.

**Architecture:** Lock-step versioning — `kg-plugin/.claude-plugin/plugin.json` `version` field matches the release tag exactly. The SKILL reads that version, the bootstrap reads that version, GoReleaser publishes that version. End-user UX: `/plugin install kg@kg-graph` → `/kg:kg-enrich` → plugin offers to install kg CLI; user accepts once; done. Dogfooders running from a checkout still work — the multi-location lookup finds `./bin/kg` before falling through to the install offer.

**Tech Stack (delta from v3):** No new Go dependencies. New surface: pure bash + jq (`bootstrap.sh`, ~120 lines), GoReleaser YAML, GitHub Actions YAML. The 4 native GHA runners (`macos-14`, `macos-13`, `ubuntu-24.04`, `ubuntu-24.04-arm`) build natively to avoid CGO cross-compile pain for tree-sitter.

**Spec:** `docs/superpowers/specs/2026-05-24-kg-bootstrap-design.md`. Re-read it before each new phase — the rationale for lock-step versioning, the rationale for `${KG_HOME:-$HOME/.config/kg}` as the install root, the rationale for AskUserQuestion vs. silent auto-install, and the rationale for 4 platforms (not 6) is not redundantly restated in each task.

**Handoff for context:** `docs/superpowers/handoffs/2026-05-24-kg-v3.1-bootstrap-handoff.md` (locks 8 scope decisions — do not relitigate; lists caveats that need to be respected during implementation).

**Prereq:** This plan builds on v3 (shipped on `main`, latest pre-v3.1 commit `2f7f9ad`). The v3 plugin already exists under `kg-plugin/` with 5 scripts + 5 tests in `kg-plugin/scripts/` and 4 commands in `kg-plugin/commands/`. `make test-scripts` currently prints 5 `OK` lines. The kg-graph marketplace wrapping at `.claude-plugin/marketplace.json` is locked due to issue #56043.

## Phase 1 — Local plumbing (no GitHub remote required)

Three tasks. Task 1 is the largest — a ~120-line bash installer plus two test files using the existing fixture-shimming pattern. Tasks 2 and 3 are markdown surgery: rewriting pre-check sections in the 4 SKILL files and the install section of `README.md`.

### Task 1: `bootstrap.sh` + happy-path + idempotency tests

A pure-bash installer that detects the platform, reads the expected version from the plugin manifest, downloads the matching release tarball + checksums from GitHub, verifies SHA-256, and extracts the 3 binaries to `$KG_HOME/bin/` while routing `manifest.json` to `$KG_HOME/extractor-plugins/tree-sitter/`. README and LICENSE inside the tarball are dropped on the floor — they're metadata, not install targets. The tests use the existing PATH-shimming fixture pattern (see `kg-plugin/scripts/tests/dump-files.test.sh`); a synthetic tarball is built in `/tmp` and a fake `curl` returns it.

The bootstrap is **idempotent**: if `$KG_HOME/VERSION` matches the expected tag and `$KG_HOME/bin/kg` is executable, it exits 0 without touching the network.

**Files:**
- Create: `kg-plugin/scripts/bootstrap.sh`
- Create: `kg-plugin/scripts/tests/bootstrap.test.sh`
- Create: `kg-plugin/scripts/tests/bootstrap-idempotent.test.sh`

**Notes for the implementer:**
- **Do not follow the spec's `tar -xzf -C bin/` pseudo-code literally.** The tarball contains README + LICENSE + manifest.json next to the 3 binaries. Extract to a tmp dir, then `mv` only the 3 binaries to `bin/` and `manifest.json` to `extractor-plugins/tree-sitter/`. The test asserts no leak.
- On macOS, `sha256sum` is absent; use `shasum -a 256` as a fallback. Both branches must be tested by virtue of the test running on whichever platform you're on.
- `$CLAUDE_PLUGIN_ROOT` is the plugin's runtime root injected by Claude Code; the test fakes it via `mktemp -d`.

- [ ] **Step 1: Create the bootstrap script**

`kg-plugin/scripts/bootstrap.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

PLUGIN_ROOT="${CLAUDE_PLUGIN_ROOT:-$(cd "$(dirname "$0")/.." && pwd)}"
KG_HOME="${KG_HOME:-$HOME/.config/kg}"
REPO_OWNER="ggfarmco"
REPO_NAME="kg"

case "$(uname -s)" in
  Darwin) OS=darwin ;;
  Linux)  OS=linux ;;
  *) echo "unsupported OS: $(uname -s) (supported: darwin, linux)" >&2; exit 2 ;;
esac
case "$(uname -m)" in
  arm64|aarch64) ARCH=arm64 ;;
  x86_64)        ARCH=amd64 ;;
  *) echo "unsupported arch: $(uname -m) (supported: arm64, amd64)" >&2; exit 2 ;;
esac
PLATFORM="${OS}_${ARCH}"

if ! command -v jq >/dev/null; then
  echo "jq is required to read plugin manifest. Install: brew install jq / apt install jq" >&2
  exit 2
fi

VERSION=$(jq -r '.version' "$PLUGIN_ROOT/.claude-plugin/plugin.json" 2>/dev/null || echo "")
if [ -z "$VERSION" ] || [ "$VERSION" = "null" ]; then
  echo "cannot read plugin.json version at $PLUGIN_ROOT/.claude-plugin/plugin.json" >&2
  exit 2
fi
TAG="v${VERSION#v}"

if [ -f "$KG_HOME/VERSION" ] \
   && [ "$(cat "$KG_HOME/VERSION")" = "$TAG" ] \
   && [ -x "$KG_HOME/bin/kg" ]; then
  echo "kg $TAG already installed at $KG_HOME"
  exit 0
fi

TMP=$(mktemp -d)
trap 'rm -rf "$TMP"' EXIT

TARBALL="kg_${TAG}_${PLATFORM}.tar.gz"
BASE_URL="https://github.com/$REPO_OWNER/$REPO_NAME/releases/download/$TAG"

echo "Downloading $BASE_URL/$TARBALL ..."
curl -fL --proto '=https' -o "$TMP/$TARBALL" "$BASE_URL/$TARBALL"
curl -fL --proto '=https' -o "$TMP/checksums.txt" "$BASE_URL/checksums.txt"

EXPECTED=$(grep " ${TARBALL}\$" "$TMP/checksums.txt" | awk '{print $1}')
if [ -z "$EXPECTED" ]; then
  echo "checksum entry missing for $TARBALL in checksums.txt" >&2
  exit 1
fi

if command -v sha256sum >/dev/null; then
  ACTUAL=$(sha256sum "$TMP/$TARBALL" | awk '{print $1}')
else
  ACTUAL=$(shasum -a 256 "$TMP/$TARBALL" | awk '{print $1}')
fi
if [ "$ACTUAL" != "$EXPECTED" ]; then
  echo "checksum mismatch for $TARBALL: expected $EXPECTED, got $ACTUAL" >&2
  echo "report at https://github.com/$REPO_OWNER/$REPO_NAME/issues" >&2
  exit 1
fi

mkdir -p "$KG_HOME/bin" "$KG_HOME/extractor-plugins/tree-sitter"

EXTRACT="$TMP/extract"
mkdir -p "$EXTRACT"
tar -xzf "$TMP/$TARBALL" -C "$EXTRACT"

for b in kg kg-extractor kg-extractor-tree-sitter; do
  if [ ! -f "$EXTRACT/$b" ]; then
    echo "tarball missing expected binary: $b" >&2
    exit 1
  fi
  mv -f "$EXTRACT/$b" "$KG_HOME/bin/$b"
  chmod +x "$KG_HOME/bin/$b"
done

if [ ! -f "$EXTRACT/manifest.json" ]; then
  echo "tarball missing expected manifest.json" >&2
  exit 1
fi
mv -f "$EXTRACT/manifest.json" "$KG_HOME/extractor-plugins/tree-sitter/manifest.json"

ln -sf "$KG_HOME/bin/kg-extractor-tree-sitter" \
       "$KG_HOME/extractor-plugins/tree-sitter/kg-extractor-tree-sitter"

echo "$TAG" > "$KG_HOME/VERSION"

echo "kg $TAG installed to $KG_HOME"
echo "Optional: export PATH=\"$KG_HOME/bin:\$PATH\""
```

Make executable:


`kg-plugin/scripts/tests/bootstrap.test.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"

PLUGIN_ROOT=$(mktemp -d)
FAKE_HOME=$(mktemp -d)
STUB=$(mktemp -d)
SHIM=$(mktemp -d)
trap 'rm -rf "$PLUGIN_ROOT" "$FAKE_HOME" "$STUB" "$SHIM"' EXIT

mkdir -p "$PLUGIN_ROOT/.claude-plugin" "$PLUGIN_ROOT/scripts"
echo '{"name":"kg","version":"0.3.1"}' > "$PLUGIN_ROOT/.claude-plugin/plugin.json"
cp ../bootstrap.sh "$PLUGIN_ROOT/scripts/"

export HOME="$FAKE_HOME"
export KG_HOME="$FAKE_HOME/.config/kg"
export CLAUDE_PLUGIN_ROOT="$PLUGIN_ROOT"

for b in kg kg-extractor kg-extractor-tree-sitter; do
  printf '#!/bin/sh\necho %s "$@"\n' "$b" > "$STUB/$b"
  chmod +x "$STUB/$b"
done
echo '{"name":"tree-sitter"}' > "$STUB/manifest.json"
echo 'fake readme'  > "$STUB/README.md"
echo 'fake license' > "$STUB/LICENSE"
tar -C "$STUB" -czf "$STUB/tarball.tar.gz" \
  kg kg-extractor kg-extractor-tree-sitter manifest.json README.md LICENSE

if command -v sha256sum >/dev/null; then
  EXPECTED_HASH=$(sha256sum "$STUB/tarball.tar.gz" | awk '{print $1}')
else
  EXPECTED_HASH=$(shasum -a 256 "$STUB/tarball.tar.gz" | awk '{print $1}')
fi

OS=$(uname -s | tr 'A-Z' 'a-z')
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
TARBALL_NAME="kg_v0.3.1_${OS}_${ARCH}.tar.gz"
echo "$EXPECTED_HASH  $TARBALL_NAME" > "$STUB/checksums.txt"

cat > "$SHIM/curl" <<EOF
#!/usr/bin/env bash
out=""
prev=""
for arg; do
  if [ "\$prev" = "-o" ]; then out="\$arg"; fi
  prev="\$arg"
done
url="\${@: -1}"
case "\$url" in
  *"$TARBALL_NAME") cp "$STUB/tarball.tar.gz" "\$out" ;;
  *"checksums.txt") cp "$STUB/checksums.txt" "\$out" ;;
  *) echo "unexpected curl URL: \$url" >&2; exit 1 ;;
esac
EOF
chmod +x "$SHIM/curl"
export PATH="$SHIM:$PATH"

bash "$PLUGIN_ROOT/scripts/bootstrap.sh" >/dev/null

fail() { echo "FAIL bootstrap.sh: $1"; exit 1; }

[ -x "$KG_HOME/bin/kg" ]                                              || fail "kg not installed"
[ -x "$KG_HOME/bin/kg-extractor" ]                                    || fail "kg-extractor not installed"
[ -x "$KG_HOME/bin/kg-extractor-tree-sitter" ]                        || fail "kg-extractor-tree-sitter not installed"
[ -f "$KG_HOME/extractor-plugins/tree-sitter/manifest.json" ]         || fail "manifest not in extractor-plugins/"
[ -L "$KG_HOME/extractor-plugins/tree-sitter/kg-extractor-tree-sitter" ] || fail "symlink missing"
[ "$(cat "$KG_HOME/VERSION")" = "v0.3.1" ]                            || fail "VERSION file wrong"

[ ! -e "$KG_HOME/bin/README.md"     ] || fail "README leaked into bin/"
[ ! -e "$KG_HOME/bin/LICENSE"       ] || fail "LICENSE leaked into bin/"
[ ! -e "$KG_HOME/bin/manifest.json" ] || fail "manifest leaked into bin/"

echo "OK bootstrap.sh"
```

If it fails: read the FAIL message. Common pitfalls:
- The `mv` from `$EXTRACT/` fails if `tar` extracted to a subdir — verify `ls "$EXTRACT"` shows the binaries at top level.
- The checksum line in `checksums.txt` uses two spaces between hash and filename (`sha256sum` format) — the bootstrap's `grep " ${TARBALL}\$"` matches a single space; that's correct because `sha256sum` outputs `<hash><space><space><filename>` and the `awk '{print $1}'` picks the first field regardless.
- On macOS, `tar` is BSD tar; the test's tarball uses GNU-compatible flags only (`-czf`, `-xzf`) so this should be portable.

`.github/workflows/release.yml`:

```yaml
name: release
on:
  push:
    tags: ['v*']

permissions:
  contents: write

jobs:
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - { os: macos-14,         goos: darwin, goarch: arm64 }
          - { os: macos-13,         goos: darwin, goarch: amd64 }
          - { os: ubuntu-24.04,     goos: linux,  goarch: amd64 }
          - { os: ubuntu-24.04-arm, goos: linux,  goarch: arm64 }
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-go@v5
        with:
          go-version: '1.26'

      - name: Build with goreleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          version: '~> v2'
          args: build --single-target --clean
        env:
          GOOS:   ${{ matrix.goos }}
          GOARCH: ${{ matrix.goarch }}
          CGO_ENABLED: '1'

      - uses: actions/upload-artifact@v4
        with:
          name: kg-${{ matrix.goos }}-${{ matrix.goarch }}
          path: dist/

  release:
    needs: build
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-go@v5
        with:
          go-version: '1.26'

      - uses: actions/download-artifact@v4
        with:
          path: dist/

      - name: Flatten artifacts for goreleaser
        run: |
          mkdir -p dist-merged
          find dist -type f \( -name 'kg' -o -name 'kg-extractor' -o -name 'kg-extractor-tree-sitter' \) -print
          ls -la dist/

      - uses: goreleaser/goreleaser-action@v6
        with:
          version: '~> v2'
          args: release --clean --skip=build
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

(The `release` job uses `--skip=build` because the binaries are already built in the matrix and downloaded via `actions/download-artifact@v4`. GoReleaser will pick them up from `dist/` and package them into the archives + checksums.txt.)

**Caveat to expect during Task 5:** the matrix-then-release pattern with `--skip=build` is the canonical GoReleaser CI shape, but the `dist/` layout from `download-artifact` differs from what GoReleaser's `release` step expects — typically the artifacts land in `dist/kg-darwin-arm64/...` instead of `dist/kg_darwin_arm64_v8.0/...`. Task 5 iterates on this; the most common fix is adding a `mv dist/kg-*/{kg,kg-extractor,kg-extractor-tree-sitter}_*/* dist/` flatten step or using `metadata.json` upload pattern. See https://goreleaser.com/customization/release/ and https://goreleaser.com/ci/actions/#fingerprinting for current patterns.
